🎯 流程指引

  1. 管理員：建立老師、學生、課程（後台現有 API）
  2. 管理員：為學生新增門禁時間（自動產生課表）
  3. 管理員：建立「上課紀錄」(Schedule/Attendance)
  4. 老師或學生：簽到 (課程一 ~ 課程四)
  5. 學生繳款：建立 Payment / StudentPermissionFee
  6. 查詢：使用下方接口檢視學生上課紀錄總覽

📊 整體流程資料流

1️⃣ 初始化階段

  • 建立老師/學生/課程: 透過後台管理界面或 API 在 tblUsers (角色為教師/學生)、tblCourse、tblCourseType、tblCourseFee 等表中新增基本資料。
  • 設定課程收費: 在 tblCourseFee 中定義課程金額、材料費、時數、拆帳比 (SplitRatio: 0-1 或百分比)。
  • 設定老師拆帳: 在 tblTeacherSettlement 中定義老師的拆帳比 (SplitRatio)，用於自動費用計算。
  • 建立教室: 在 tblClassroom 中新增教室資訊，供課表安排使用。

2️⃣ 權限授予階段

  • 新增學生門禁時間 (tblStudentPermission): 指定學生、老師、課程、教室、上課日期/時間 (DateFrom~DateTo)、星期 (Days: 1-7)、課表模式 (ScheduleMode: 1=週固定,2=雙週,3=單次)、上課模式 (CourseMode: 1=現場,2=視訊)。系統自動根據週期設定在 tblSchedule 中生成課程表。
  • 建立學生權限費用 (tblStudentPermissionFee): 每筆 StudentPermission 在新增時建立對應的費用記錄，記錄該期課程的「應收金額」(TotalAmount)、「課程時數」(Hours)、「繳款期限」(PaymentDueDate)。一個 StudentPermission 可對應多筆 StudentPermissionFee (按課程週期分組，例如每月一期)。

3️⃣ 簽到紀錄階段

  • 建立上課紀錄 (tblAttendance): 老師或管理員為每堂課建立簽到記錄，記錄上課日期、類型 (1=出席, 0=缺席, 2=請假)、修改者 (ModifiedUserId)、修改時間。
  • 自動計算簽到費用 (tblAttendanceFee): 系統為每筆 Attendance 自動建立對應的 tblAttendanceFee，記錄：
    — Hours (實際上課時數，例如 1.5 小時，依 Attendance.Type 與課程設定判斷)
    — Amount (單堂學費，即該次上課應收的金額，已含拆帳比計算)
    — AdjustmentAmount (增減金額，可由人工調整，正數為增加、負數為減少)
    — UseSplitRatio (實際使用的拆帳比，先正規化課程拆帳比與老師拆帳比 [若>1則÷100]，再取最小值 [若都無則為0]，最後限制在 0~1)
    — SourceHoursTotalAmount (原始時數總金額 = 該次上課的基礎費用，未含 AdjustmentAmount，用於追蹤原始計算值)
    — 費用計算邏輯：最終費用 = Amount + AdjustmentAmount
  • 分組對應: 系統依 StudentPermission 的所有 Attendance 日期排序，多筆簽到累積對應至一筆 StudentPermissionFee，用以判斷該期簽到是否已繳款。

4️⃣ 繳款管理階段

  • 建立繳款記錄 (tblPayment): 當學生繳款時，在 tblPayment 中建立一筆記錄，記錄「繳款日期」(PayDate)、「繳款金額」(Pay)、「折扣」(DiscountAmount)、「備註」(Remark)，並與對應的 StudentPermissionFee 關聯 (1:1 唯一對應)。系統自動產生「收據編號」(ReceiptNumber)，格式為 {民國年:3位}B{月份:2位}{序號:4位} (例如 114B060091)，每月獨立編號。
  • 更新費用狀態: StudentPermissionFee 記錄繳款後，其 Payment 欄位被填充，TotalActualAmount 同時更新為繳款金額 - 折扣。系統以此判斷該期課程是否「已繳費」。
  • 費用與簽到對應: 同一 StudentPermission 的多筆簽到 (tblAttendance) 累積費用後，對應至一筆 StudentPermissionFee，當該費用被繳款時 (tblPayment)，即代表該組簽到已完成結算。一個 StudentPermission 可產生多筆 StudentPermissionFee (按課程週期或手動分組)，每筆獨立計費與繳款。

5️⃣ 查詢與報表階段

  • 學生權限列表查詢: 透過 GET /api/v1/StudentAttendance/{userId} 查詢特定學生的所有已授予權限及其簽到摘要，包括「總簽到次數」、「已繳費期數」、「未繳費期數」等統計。
  • 簽到詳細查詢: 透過 GET /api/v1/StudentAttendance/Detail/{studentPermissionId} 查詢特定權限的所有簽到記錄及對應的 tblAttendanceFee，並顯示該權限的所有 StudentPermissionFee 與繳款狀態。
  • 單堂費用詳情: 透過介面查詢單堂簽到的詳細費用計算過程，包括拆帳比、基礎金額、時數、調整金額等。
  • 上課拆帳明細表 (PDF): 透過 GET /api/pdf/v1/SalaryReport 依老師與日期範圍統計所有簽到金額、時數，聚合成每位老師的薪資表、應付金額、稅前稅後等。
  • 公司獲利彙總表 (PDF): 透過 GET /api/pdf/v1/CompanyProfitSummary 按老師分組統計：實收學費 (基於 tblPayment)、應付薪資、毛利、淨收入。未繳費判斷基於 StudentPermissionFee.Payment 關聯是否填充。加總後若有小數，毛利/淨收入對外輸出採無條件進位 (ceil)。
  • 營業日總表 (PDF): 透過 GET /api/pdf/v1/DailyReport 生成指定日期的營業日總表，含零用金管理、簽到統計、營業收入明細。

6️⃣ 退款管理階段

  • 查詢退款資訊: 透過 GET /api/v1/StudentRefund/{studentPermissionFeeId} 取得單筆費用的退款摘要，包括應收金額、已收金額、折扣、已退金額、收據編號、退款日期。資料來源涵蓋 tblStudentPermissionFee、tblPayment、tblRefund 的聯合查詢。
  • 新增/更新退款 (tblRefund): 透過 POST /api/v1/StudentRefund 建立或更新退款記錄，包括退款日期、退款金額、退款原因、結帳單號。遵循與 tblPayment 相同的收據編號規則 (114B{月}{序號})，寫入 tblRefund，並自動記錄到 tblAuditLog。
  • 金額關聯: 退款不改變原繳款紀錄 (tblPayment)，但系統報表在計算「實收/未收」與「淨利潤」時會自動一併考量退款金額 (實收 = Pay - RefundAmount)。
  • 1:1 對應: StudentPermissionFee 與 Refund 保持 1:1 關聯 (若存在)，支援單筆費用完全或部分退款。

7️⃣ 日終關帳流程 ⭐ 新增自動化

  • 查詢當日簽到狀態: 透過 GET /api/v1/CloseAccount/DailyStatus/{date} 回傳當日全校課表清單、每堂課的簽到狀況、統計資訊 (課程總數、已簽到、未簽到) 與是否可關帳 (CanCloseAccount) 標誌。
  • 批量簽到（可選）: 透過 POST /api/v1/CloseAccount/CheckInAll/{date} 將當日所有未簽到的課程自動建立 tblAttendance 與對應的 tblAttendanceFee。費用計算遵循標準邏輯 (基於 CourseFee 與拆帳比自動計算)，並寫入 tblAuditLog 記錄該批操作。
  • 儲存關帳 (tblCloseAccount): 透過 POST /api/v1/CloseAccount 在所有簽到完成後儲存關帳記錄。輸入日期與提存金額 (DepositAmount)。系統自動計算：
    — 昨日零用金結餘 (YesterdayPettyIncome)：取前一日 tblCloseAccount.PettyIncome
    — 今日營業收入 (BusinessIncome)：當日所有 tblPayment 的 Pay 加總 + DiscountAmount 總和
    — 關帳結算金額 (CloseAccountAmount = YesterdayPettyIncome + BusinessIncome)
    — 零用金結餘 (PettyIncome = CloseAccountAmount − DepositAmount)
    並寫入 tblCloseAccount 與 tblAuditLog。
  • 查詢關帳資料: 透過 GET /api/v1/CloseAccount/Detail/{date} 取得單日完整關帳內容；透過 GET /api/v1/CloseAccount (可帶 startDate, endDate 參數) 查詢日期範圍內的關帳列表，支援分頁。
  • 營業日總表 PDF 匯出: 透過 GET /api/pdf/v1/DailyReport/{date} 生成營業日總表 PDF，含關帳資料、簽到明細、零用金管理統計，可用於會計對帳。

8️⃣ 軟刪除與審計

  • 軟刪除機制 (IsDelete): tblStudentPermission、tblAttendance、tblStudentPermissionFee、tblPayment、tblRefund 等核心表都使用 IsDelete 欄位 (布林值) 進行邏輯刪除，而非物理刪除，確保資料完整可溯源。查詢時預設篩選 IsDelete=false。
  • 審計日誌 (tblAuditLog): 所有重要操作均記錄在審計表中，包括新增/編輯/刪除課程、建立/修改費用、簽到、繳款、退款、關帳等。每筆審計記錄包含：TableName (表名)、RecordId (記錄ID)、OperationType (操作類型)、OldValue/NewValue (JSON 格式)、OperationTime (時間戳)、OperationUserId (操作人)、IPAddress (來源IP)。
  • 歷史追蹤: tblAttendanceFee 與 tblStudentPermissionFee 中的 CourseSplitRatio、TeacherSplitRatio 欄位保留當時快照值，支援費用計算邏輯變更後仍能追溯歷史。

💾 核心資料表關係圖

【使用者與課程基礎表】
tblUser (使用者：老師/學生)
    └─ 1:1 → tblTeacherSettlement (老師結算：拆帳比、薪資統計)
tblCourse (課程)
    └─ 1:1 → tblCourseFee (課程收費：金額、材料費、拆帳比)
tblCourseType (課程類別)
tblClassroom (教室)

【權限與課表】
tblStudentPermission (學生授課權限)
    ├─ FK: userId → tblUser (學生)
    ├─ FK: teacherId → tblUser (老師)
    ├─ FK: courseId → tblCourse
    ├─ FK: classroomId → tblClassroom
    └─ 自動產生課表:
        └─ 1:N → tblSchedule (課程表：按週期產生)
    └─ 1:N → tblAttendance (簽到記錄)
        └─ 1:1 → tblAttendanceFee (簽到費用：時數、金額、增減、拆帳比)

【費用與繳款】
tblStudentPermission
    └─ 1:N → tblStudentPermissionFee (權限費用：按週期/課程組計費)
        └─ 1:1 → tblPayment (繳款記錄：一對一關聯繳款)
        └─ 1:1 → tblRefund (退款記錄：一對一關聯退款)
        └─ 關聯多筆 tblAttendance (每組簽到對應一筆 StudentPermissionFee)

【審計】
tblAuditLog (審計日誌：記錄所有重要操作)

【結算】
tblCloseAccount (關帳記錄：每日零用金管理)
